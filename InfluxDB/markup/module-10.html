Phase 3: Storage & Retention

3 Multi-Tier Retention Strategy
Tier 1: Raw Data

7 days retention - 1-second precision for real-time monitoring and debugging

Tier 2: Hourly Aggregates

90 days retention - Hourly averages for trend analysis and capacity planning

Tier 3: Daily Summaries

1 year retention - Daily statistics for long-term historical analysis

Downsampling Task Example
// Hourly Downsampling Task
option task = {
    name: "Hourly Downsampling",
    every: 1h,
    offset: 5m
}

from(bucket: "system_metrics_raw")
    |> range(start: -1h)
    |> filter(fn: (r) => r._measurement == "system_metrics")
    |> aggregateWindow(
        every: 1h,
        fn: mean,
        createEmpty: false
    )
    |> to(bucket: "system_metrics_1h", org: "myorg")

Phase 4: Querying & Analytics

4 Essential Queries
Current System Status
from(bucket: "system_metrics_raw")
    |> range(start: -5m)
    |> filter(fn: (r) => r._measurement == "system_metrics")
    |> filter(fn: (r) => 
        r._field == "cpu_usage_percent" or 
        r._field == "memory_usage_percent"
    )
    |> last()
    |> pivot(rowKey:["_time"], 
             columnKey: ["_field"], 
             valueColumn: "_value")
    |> group(columns: ["host"])
24-Hour Trend Analysis
from(bucket: "system_metrics_raw")
    |> range(start: -24h)
    |> filter(fn: (r) => r._measurement == "system_metrics")
    |> filter(fn: (r) => r._field == "cpu_usage_percent")
    |> aggregateWindow(every: 1h, fn: mean)
    |> yield(name: "hourly_cpu_average")
Anomaly Detection
// Calculate z-score for anomaly detection
data = from(bucket: "system_metrics_raw")
    |> range(start: -7d)
    |> filter(fn: (r) => r._field == "cpu_usage_percent")
    |> aggregateWindow(every: 5m, fn: mean)

// Identify values beyond 2 standard deviations
data
    |> map(fn: (r) => ({
        r with
        z_score: (r._value - mean) / stddev,
        is_anomaly: if abs(z_score) > 2.0 
                    then 1 else 0
    }))
    |> filter(fn: (r) => r.is_anomaly == 1)

Statistical outlier detection

Missing data detection

Alert Check Example
// High CPU Alert
option task = {
    name: "Alert: High CPU Usage",
    every: 1m
}

from(bucket: "system_metrics_raw")
    |> range(start: -5m)
    |> filter(fn: (r) => r._field == "cpu_usage_percent")
    |> mean()
    |> monitor.check(
        crit: (r) => r._value >= 90.0,
        warn: (r) => r._value >= 70.0,
        messageFn: (r) => "CPU: ${r._value}% on ${r.host}"
    )
    |> monitor.write(bucket: "alerts_history")
