Example: CPU Usage Panel

InfluxQL Query 
SELECT 100 - mean("usage_idle") AS "cpu_usage" 
FROM "cpu" 
WHERE $timeFilter 
  AND "cpu" = 'cpu-total' 
  AND "host" =~ /^$host$/
GROUP BY time($__interval), "host" 
FILL(null)

Module 3: Monitoring InfluxDB (15 min)

Internal Metrics Overview

Internal Database

InfluxDB exposes metrics via the _internal database (v1.x) or _monitoring bucket (v2.x)

Key Metrics to Monitor
Write Performance
  • Write request rate
  • Points per second
  • Write errors
Query Performance
  • Query execution rate
  • Average duration
  • Query errors
Memory & Cache
  • Cache size
  • Cache hit ratio
  • Memory usage
Storage
  • Disk usage
  • Shard statistics
  • Series cardinality
Example Monitoring Queries
Write Throughput 
-- Points written per second
SELECT derivative(mean("pointReq"), 1s) AS "points_per_sec"
FROM "_internal"."monitor"."write"
WHERE time > now() - 1h
GROUP BY time(1m)
Query Duration 
-- Average query duration (milliseconds)
SELECT mean("queryDurationNs") / 1000000 AS "avg_duration_ms"
FROM "_internal"."monitor"."queryExecutor"
WHERE time > now() - 1h
GROUP BY time(1m)
Telegraf Configuration
telegraf.conf 
[[inputs.influxdb]]
  urls = ["http://localhost:8086/debug/vars"]
  timeout = "5s"

[[inputs.cpu]]
  percpu = true
  totalcpu = true

[[inputs.mem]]

[[inputs.disk]]
  mount_points = ["/var/lib/influxdb"]

[[outputs.influxdb_v2]]
  urls = ["http://localhost:8086"]
  token = "$INFLUX_TOKEN"
  organization = "my-org"
  bucket = "monitoring"

Module 4: Alerting Systems (15 min)

Alert Design Principles

CRITICAL

Immediate action required

Example: Database down

WARNING

Attention needed soon

Example: Disk > 80%

INFO

Informational only

Example: Backup complete

Grafana Alert Configuration

Alert Rule JSON 
{
  "name": "High CPU Usage",
  "conditions": [{
    "type": "query",
    "evaluator": {
      "type": "gt",
      "params": [80]
    }
  }],
  "executionErrorState": "alerting",
  "frequency": "1m",
  "for": "5m",
  "message": "CPU usage above 80% for 5 minutes",
  "notifications": [
    {"uid": "slack-channel"}
  ]
}
Notification Channels
Slack

Team chat alerts

Email

Email notifications

PagerDuty

On-call paging

Custom integrations

Alert Best Practices
  • Make alerts actionable with clear next steps
  • Use appropriate severity levels
  • Add evaluation periods to prevent false alarms
  • Include context in alert messages
  • Test alerts before production deployment
  • Document response procedures
  • Review and tune alert thresholds regularly
