What is InfluxDB?

Time-Series Database

InfluxDB is an open-source time-series database (TSDB) designed specifically for handling time-stamped data with high write and query loads. It's optimized for use cases like IoT sensors, DevOps monitoring, and real-time analytics.

High Performance

Handle 100K-1M writes per second with optimized storage engine

Compression

10-100x compression ratios with specialized algorithms

SQL-Like Queries

Familiar query languages: InfluxQL and Flux

Core Components

Understanding the Data Model
Line Protocol Format:
Format
<measurement>[,<tag_key>=<tag_value>[,<tag_key>=<tag_value>]] <field_key>=<field_value>[,<field_key>=<field_value>] [<timestamp>]
Example:
Sample Data
weather,location=us-midwest,season=summer temperature=82,humidity=71 1465839830100400200

Breakdown:

  • Measurement: weather
  • Tags: location=us-midwest, season=summer
  • Fields: temperature=82, humidity=71
  • Timestamp: 1465839830100400200 (nanoseconds)
Tags (Indexed)
Characteristics:
  • Stored as strings
  • Indexed for fast queries
  • Used for filtering (WHERE)
  • Used for grouping (GROUP BY)
Best For:
  • Host names
  • Regions/Locations
  • Sensor IDs
  • Status codes
Low cardinality data
Fields (Not Indexed)
Characteristics:
  • Can be different types (float, int, string, bool)
  • Not indexed
  • Actual measured values
  • Required (at least one)
Best For:
  • Temperature readings
  • CPU usage percentages
  • Response times
  • Measurements/Metrics
High cardinality is OK

Understanding Series Cardinality

Series = Unique combination of measurement + tag set

Example Series
# Series 1
cpu,host=server01,region=us-east value=45.2

# Series 2
cpu,host=server02,region=us-east value=52.1

# Series 3 (different because of region tag)
cpu,host=server01,region=us-west value=38.7
Cardinality Calculation:

Total Series = Measurements × (Tag1_Values × Tag2_Values × ... × TagN_Values)

Important:

High cardinality (millions of series) can impact performance. Keep series count under 1M for optimal performance.

InfluxDB OSS vs Cloud

InfluxDB OSS (Open Source)
Advantages:
  • Free and open source
  • Self-hosted (full control)
  • No usage limits
  • Data privacy/security
Considerations:
  • Manual scaling
  • Self-managed backups
  • Infrastructure overhead
  • No built-in clustering (OSS)
Best for: On-premise, custom integrations, learning
InfluxDB Cloud
Advantages:
  • Fully managed
  • Automatic scaling
  • Built-in monitoring
  • Global availability
Considerations:
  • Usage-based pricing
  • Vendor lock-in
  • Internet dependency
  • Limited customization
Best for: Quick deployment, scalable workloads, minimal ops

Quick Start: Write Your First Data Point

Follow these steps to write and query your first data in InfluxDB:

Step 1: Start InfluxDB (Docker)
Terminal
docker run -d -p 8086:8086 \
  --name influxdb \
  influxdb:2.7
Step 2: Access the UI

Open http://localhost:8086 in your browser

Step 3: Complete Setup
  • Username: admin
  • Organization: myorg
  • Bucket: mybucket
Step 4: Write Data
Using curl
curl -XPOST "http://localhost:8086/api/v2/write?org=myorg&bucket=mybucket" \
  -H "Authorization: Token YOUR_TOKEN" \
  --data-binary 'temperature,location=room1 value=22.5'
Step 5: Query Data
Flux Query
from(bucket: "mybucket")
  |> range(start: -1h)
  |> filter(fn: (r) => r._measurement == "temperature")

Docker Installation
Pull the InfluxDB image:
docker pull influxdb:2.7
Run the container:
docker run -d -p 8086:8086 \
  -v influxdb2-data:/var/lib/influxdb2 \
  -v influxdb2-config:/etc/influxdb2 \
  --name influxdb \
  influxdb:2.7
Access the UI:

Open http://localhost:8086

Ubuntu/Debian Installation
Add the repository:
wget -q https://repos.influxdata.com/influxdata-archive_compat.key
sudo apt-get update
sudo apt-get install influxdb2
Start the service:
sudo systemctl start influxdb
sudo systemctl enable influxdb
macOS Installation
Install using Homebrew:
brew install influxdb
Start the service:
brew services start influxdb
