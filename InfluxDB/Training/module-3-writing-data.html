Methods of Data Ingestion

Multiple Ingestion Paths

InfluxDB supports multiple methods for writing data. Choose the method that best fits your architecture.

Line Protocol Format

The universal format for all write methods:

measurement[,tag=value] field=value [timestamp]
Example:
temperature,location=room1 value=22.5 1633024800000000000

Batch Writes & Performance

Performance Critical

Batch writing is essential for high-performance ingestion. Writing individual points is inefficient.

Python Batch Example 
from influxdb_client import InfluxDBClient, Point

client = InfluxDBClient(url="http://localhost:8086", 
                        token="YOUR_TOKEN", org="myorg")
write_api = client.write_api()

# Create batch
points = []
for i in range(5000):
    point = Point("temperature")\
        .tag("location", f"room{i}")\
        .field("value", 20 + i * 0.01)
    points.append(point)

# Write batch
write_api.write(bucket="mybucket", record=points)
print(f"Written {len(points)} points")

Ingestion Best Practices

DO: Optimize for High Throughput
  • Use batch writes - 1000-5000 points per batch
  • Implement buffering - Buffer points in memory
  • Parallel writes - Use multiple threads
  • Async writes - Don't block on acknowledgment
DON'T: Common Mistakes
  • Single point writes - Kills performance
  • Unbounded batches - Can cause memory issues
  • No error handling - Lose data on failures
  • Ignore backpressure - Can overwhelm InfluxDB
