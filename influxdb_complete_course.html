Module 4: Design Insights & Schema Optimization

Topics: TSM storage engine, TSI index, schema design, series cardinality, compression

Key Metrics:

  • Timestamp compression: 10:1 ratio (delta-of-delta)
  • Float compression: 3:1 ratio (Gorilla algorithm)
  • Overall: ~5:1 compression typical

Key Learning: Internal architecture, optimization strategies, cardinality management

Module 8: Influx Query Language (InfluxQL)

Topics: SELECT queries, aggregations, GROUP BY, continuous queries, optimization

SELECT MEAN(value) 
FROM temperature 
WHERE time > now() - 1h AND location='room1'
GROUP BY time(5m)

CREATE CONTINUOUS QUERY "cq_30m" ON "mydb"
BEGIN
  SELECT MEAN(value) INTO "avg_temp" FROM "temperature"
  GROUP BY time(30m), *
END

Key Learning: SQL-like querying, time-based aggregations, pre-aggregation with CQs

INTERMEDIATE

Module 9: Flux Language

Topics: Functional programming, transformations, joins, geotemporal data, Flux tasks

from(bucket: "mydb/autogen")
  |> range(start: -1h)
  |> filter(fn: (r) => r._measurement == "temperature")
  |> mean()
  
// Join example
temp = from(bucket: "mydb") |> range(start: -1h)
humidity = from(bucket: "mydb") |> range(start: -1h)
join(tables: {temp: temp, humidity: humidity}, on: ["_time"])

Key Learning: Advanced transformations, multi-source queries, functional data scripting
